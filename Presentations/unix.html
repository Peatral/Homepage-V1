Nochfolgende Informationen sind für Linux

Moritz hat sich stark vertan
  • Monolithischer Kernel
  • Treiber und Kernel-Module laufen im privilegierten Modus
  • Einige Module laufen im eingeschränkten Modus
  • Viele Treiber können als Modul vom System dynamisch nachgeladen werden
  • Im System laufende Programme bekommen vom Kernel Prozessorzeit zugewiesen
  • Jeder Prozess bekommt einen eigenen geschützen Speicherbereich
  • Jeder Prozess kann nur über Systemaufrufe auf Gerätetreiber und Betriebssystem zugreifen
  • Prozess läuft dabei im Benutzermodus, Kernel im Kernelmodus
  • Direkte Zugriffe sind nur selten unter bestimmten Bedingungen erlaubt
  • -> Absicherung gegen Systemabstürze
  • Immer mehr Einfluss von Mikrokerneln

Probleme der Unix-Sicherheit

  • Zugriffsberechtigungen
  • SSH-keys
  • Verwaltung der Anmeldedaten für root-Accounts
  • Verfolgung der Benutzeraktivitäten
  • Verwaltung von an Dritte abgegebene Superuser-Anmeldedaten
  • Effektivität/Sicherheit von Jump-Servern
